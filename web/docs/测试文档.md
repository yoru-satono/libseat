**日期**：2026-06-02

## 1. 测试策略

### 1.1 分层测试

```
E2E 测试（Playwright）
  ↓  关键用户流程，对接真实后端
单元测试（Vitest）
  ↓  Store 逻辑、HTTP 拦截器、独立组件
```

| 层次 | 工具 | 测试范围 | 文件位置 |
|------|------|---------|---------|
| 单元测试 | Vitest 4.1 + @vue/test-utils 2.4 | Store、HTTP 层、通用组件、工具函数 | `src/**/__tests__/*.spec.ts` |
| E2E 测试 | Playwright 1.59 | 7 个用户/管理员核心流程 | `e2e/*.spec.ts` |

### 1.2 测试原则

- E2E **不使用 MSW**，直接对接本地后端（`http://localhost:8080`），数据来自 `02_seed.sql` 种子数据
- 单元测试对 API 层使用 `vi.mock` 隔离网络请求
- 每个单元测试在独立 Pinia 实例中运行（`setActivePinia(createPinia())`），beforeEach 清空 localStorage
- E2E 每个测试拥有独立浏览器 Context，localStorage 天然隔离，无需手动清除
- 数据依赖型用例（需特定 DB 状态）使用 `test.skip()` 条件跳过，不强制失败

## 4. 已知测试注意事项

### 4.1 测试数据隔离

E2E 对接真实数据库，**每次运行会写入真实数据**（如创建图书馆、创建预约）。

- 预约用例使用 `Date.now()` 派生动态时段，避免重复运行时「时段冲突」错误
- 图书馆用例创建的记录名称带 `_${Date.now()}` 后缀，每次唯一
- 锁定用户操作会修改 DB 状态，若锁定后未手动解锁，下次运行时该用户处于 LOCKED 状态（可能影响登录）

### 4.2 Headless UI 交互

页面中的下拉选择器（图书馆筛选、楼层筛选等）使用 Headless UI `Listbox`，**非**原生 `<select>`。Playwright 脚本不能使用 `selectOption()`，必须：

```typescript
// 正确做法
await page.locator('button').filter({ hasText: '全部图书馆' }).click()
await page.locator('[role="option"]').first().click()

// 错误做法（会找不到元素）
await page.locator('select').selectOption(...)
```

### 4.3 元素定位依赖

以下页面特性是 E2E 测试可靠运行的前提，**不得随意修改**：

| 元素 | 属性 | 用途 |
|------|------|------|
| LoginPage 用户名输入框 | `id="userNo"` + `<label for="userNo">` | `getByLabel('学号 / 工号')` |
| LoginPage 密码输入框 | `id="password"` + `<label for="password">` | `getByLabel('密码')` |
| SeatCard 根元素 | `data-testid="seat-card"` | `locator('[data-testid="seat-card"]')` |
| LibrariesPage 编辑按钮 | `aria-label="编辑"` | `getByRole('button', {name:'编辑'})` |
| LibrariesPage 删除按钮 | `aria-label="删除"` | `getByRole('button', {name:'删除'})` |
| LibrariesPage 名称输入框 | `id="lib-name"` + `<label for="lib-name">` | `getByLabel('名称')` |
| LibrariesPage 地址输入框 | `id="lib-address"` + `<label for="lib-address">` | `getByLabel('地址')` |

### 4.4 浏览器要求

当前配置使用 Playwright 内置 Chromium（`projects: [{ name: 'chromium', use: { ...devices['Desktop Chrome'] } }]`）。安装命令：`npx playwright install --with-deps chromium`。如需在其他环境中运行（如 Ubuntu 较新版本不被 Playwright 支持），可使用 distrobox 创建 Ubuntu 24.04 容器安装浏览器。
