# LibSeat 前端概要设计文档

**项目**：LibSeat 图书馆座位预约系统 — 前端  
**技术栈**：Vue 3 + TypeScript + Vite  
**版本**：1.1.0  
**日期**：2026-06-02

---

## 1. 系统概述

LibSeat 前端是图书馆座位预约系统的 Web 客户端，以 SPA（单页应用）形式交付。系统对接 Spring Boot 后端（`http://localhost:8080/api/v1`），覆盖普通用户预约流程和管理员后台两套交互界面。

### 1.1 设计目标

- **双端适配**：同一套代码兼容桌面浏览器和移动端，无需独立 App
- **响应迅速**：Vite 构建 + 代码分割，初始包体积控制在合理范围
- **安全可靠**：JWT 双 Token 自动刷新，路由守卫阻止越权访问
- **可测试性**：分层架构使 Store、HTTP 层、组件均可独立单元测试

前端使用 Tailwind CSS v4 的 `@theme` + `@utility` 构建完整设计 Token 体系。

**色彩系统（OKLCH 色彩空间）**

| Token | 色相 | 用途 |
|-------|------|------|
| `primary` (50–900) | 深海军蓝 (hue 265) | 主按钮、链接、活跃状态、导航栏 |
| `accent` (50–900) | 暖陶土色 (hue 30) | 强调按钮、统计卡片、续约操作 |
| `success` (50–700) | 鼠尾草绿 (hue 155) | 成功状态、已签到、已批准 |
| `warning` (50–700) | 暖琥珀 (hue 85) | 警告状态、爽约、待审核 |
| `danger` (50–700) | 柔和玫瑰红 (hue 15) | 危险操作、已取消、已锁定 |
| `surface` (50–900) | 暖灰中性色 (hue 265) | 背景、卡片、边框、文字 |

**排版**

| Token | 字体栈 | 用途 |
|-------|--------|------|
| `font-display` | Noto Serif SC, Georgia, serif | 页面标题、Logo、区段标题 |
| `font-body` | Inter, Noto Sans SC, PingFang SC, sans-serif | 正文、表单、表格 |
| `font-mono` | JetBrains Mono, Fira Code, monospace | 代码、学号等标识符 |

**语义化工具类**（通过 `@utility` 定义）

| 类名 | 用途 |
|------|------|
| `card` / `card-hover` | 统一卡片样式（白色背景、圆角、边框），hover 悬停提升 |
| `input-field` | 统一输入框样式（边框、聚焦环、禁用态） |
| `page-heading` / `section-heading` | 页面/区段标题（font-display + 相应字号） |
| `skeleton` | 骨架屏加载占位（带动画 shimmer 效果） |
| `animate-fade-in-up` 等 | 入场动画（fade-in、scale-in、shake 等） |
| `stagger-1` 至 `stagger-10` | 列表交错动画延迟 |

**动画**

- 页面切换：`<Transition name="page" mode="out-in">` 实现 fade + slide
- 通知铃铛：新通知时触发 CSS shake 动画
- 按钮 Loading：内联 SVG spinner（替代文字变化）
- 卡片列表：基于索引的 stagger 交错淡入

## 3. 模块划分

### 3.1 认证模块

负责用户身份识别与会话管理。

- **LoginPage**：学号/工号 + 密码登录
- **RegisterPage**：新用户注册（含邮箱验证）
- **ActivatePage / EmailConfirmPage**：邮件 token 激活
- **ForgotPasswordPage / ResetPasswordPage**：密码找回

### 3.2 用户功能模块

普通用户（学生/教师）的核心操作。

| 子模块 | 功能 |
|--------|------|
| 首页 | 今日预约状态、图书馆快捷入口 |
| 座位浏览 | 筛选（图书馆/日期/时间段/楼层/区域/设施）、匿名可访问 |
| 预约管理 | 查看我的预约列表（分 Tab 按状态筛选）、取消、续约 |
| 等待队列 | 查看我的等待记录、状态跟踪 |
| 通知中心 | 未读/已读、全部标已读、通知删除 |
| 个人资料 | 查看/修改基本信息、提交信息修改申请、导出记录 |

### 3.3 管理员模块

后台管理，仅 `role=ADMIN` 的用户可访问。

| 子模块 | 功能 |
|--------|------|
| 仪表盘 | 今日统计卡片 |
| 用户管理 | 搜索、锁定/解锁、重置密码 |
| 预约管理 | 全平台预约列表、状态筛选 |
| 修改申请审核 | 批准/拒绝用户信息修改申请 |
| 图书馆管理 | 新增/编辑/删除图书馆 |
| 座位管理 | 新增/编辑/删除座位 |
| 系统规则 | 查看/修改预约规则 |
| 审计日志 | 只读，查看管理员操作记录 |

## 4. 路由设计

### 4.1 路由分组与 meta

| 分组 | 路径 | Layout | 守卫 |
|------|------|--------|------|
| 认证 | `/login` `/register` `/activate` 等 | AuthLayout | `guest: true`（已登录跳首页） |
| 公开用户 | `/` `/seats` | UserLayout | 无 |
| 登录用户 | `/reservations` `/profile` 等 | UserLayout | `requiresAuth: true` |
| 管理员 | `/admin/**` | AdminLayout | `requiresAdmin: true` |

### 4.2 路由守卫逻辑

```
beforeEach:
  1. 若有 token 但无 currentUser → fetchCurrentUser()（刷新页面恢复会话）
  2. requiresAdmin → 未登录跳 /login；非管理员跳 /
  3. requiresAuth  → 未登录跳 /login
  4. guest         → 已登录跳首页或 /admin
```

### 4.3 未匹配路由

所有 `/:pathMatch(.*)*` 重定向到 `/`，不展示 404 页面。

## 5. 认证与安全

### 5.1 双 Token 方案

| Token | 有效期 | 存储 | 用途 |
|-------|--------|------|------|
| accessToken | 2 小时 | localStorage | 每次请求 Authorization 头 |
| refreshToken | 7 天 | localStorage | 刷新 accessToken |

### 5.2 自动刷新流程

```
请求返回 A0201（access token 过期）
    ↓
isRefreshing = false？
  ├─ 是：发起 POST /auth/refresh
  │       ├─ 成功：更新两个 token，清空等待队列（重试各原请求）
  │       └─ 失败：清空 localStorage，跳转 /login
  └─ 否：入队 failedQueue，等待刷新完成后重试
```

并发请求同时过期时，只发一次刷新请求（队列合并机制）。

### 5.3 强制登出条件

- `A0202`：refreshToken 也过期或无效
- `A0100`：未登录状态下访问受保护接口

## 6. 状态管理

### 6.1 useAuthStore

```
state:  accessToken | refreshToken | currentUser
getter: isAuthenticated | isAdmin
action: login | logout | fetchCurrentUser | setTokens | clearTokens
持久化: tokens → localStorage（应用启动时同步读取）
```

### 6.2 useNotificationStore

```
state:  unreadCount
action: fetchUnreadCount | startPolling | stopPolling
轮询:   登录后每 60 秒调 GET /notifications/unread-count，更新导航栏徽章
```

## 8. 接口对接规范

### 8.1 Base URL

```
开发环境: http://localhost:8080/api/v1  （由 VITE_API_BASE_URL 环境变量配置）
```

### 8.2 统一响应结构

```typescript
interface ApiResponse<T> {
  code: string      // "00000" 为成功，其余为业务错误
  message: string
  data: T
  timestamp: string
}
```

### 8.3 HTTP 状态码

后端所有接口统一返回 HTTP 200，业务成功与否通过 `code` 字段判断。前端不依赖 HTTP 状态码做业务分支。

### 8.4 分页参数

- 请求：`page`（从 1 开始）+ `pageSize`（默认 20，最大 100）
- 响应：`PageResult<T>`，含 `items` / `total` / `page` / `pageSize` / `totalPages`

## 10. 关键约束

1. **不使用 SSR**：纯 SPA，所有路由由前端 History API 处理，Nginx 需配置 `try_files`。
2. **不使用 MSW**：E2E 测试直接对接真实后端，不引入 Mock Service Worker。
3. **不依赖 Vuex**：全量使用 Pinia，无 Vuex 残留。
4. **Tailwind v4 配置**：通过 `@tailwindcss/vite` 插件集成，无传统 `tailwind.config.js`。使用 `@theme` 定义设计 Token（6 组色阶：primary/accent/success/warning/danger/surface），使用 `@utility` 定义语义化工具类（card、input-field、page-heading 等）。
