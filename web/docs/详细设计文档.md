## 1. API 层

### 1.1 Axios 实例（`src/api/http.ts`）

#### 实例配置

```typescript
const http = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8080/api/v1',
  headers: { 'Content-Type': 'application/json' },
  timeout: 15000,
})
```

#### 请求拦截器

读取 `localStorage.accessToken`，存在则注入 `Authorization: Bearer {token}` 头。

#### 响应拦截器

| 响应体 code | 处理逻辑 |
|------------|---------|
| `00000` | 直接返回 response |
| `A0201` | 触发 token 刷新流程（见下文） |
| `A0202` / `A0100` | `clearAuthAndRedirect()`：清空 localStorage → `window.location.href = '/login'` |
| 其余业务错误 | `throw new ApiError(code, message)` |
| `responseType = 'blob'` | Blob 响应直接透传，跳过 code 检查 |

#### Token 刷新队列机制

模块级变量：`isRefreshing: boolean`、`failedQueue: QueueItem[]`。

```
收到 A0201：
  ├─ isRefreshing = true？→ 将原请求 resolve/reject 推入 failedQueue，等待
  └─ isRefreshing = false？
       标记 _retry=true，设 isRefreshing=true
       POST /auth/refresh（用独立 axios 实例，不经过拦截器）
         成功：localStorage 写新 tokens → processQueue(null, newToken) → 重试原请求
         失败：processQueue(err) → clearAuthAndRedirect()
       finally: isRefreshing = false
```

#### ApiError 类

```typescript
class ApiError extends Error {
  constructor(public code: string, message: string) { super(message) }
}
```

业务层通过 `instanceof ApiError` 区分业务错误与网络错误。

## 2. 状态管理（Pinia）

### 2.1 useAuthStore（`src/stores/auth.ts`）

#### State

| 字段 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `accessToken` | `string \| null` | `localStorage.getItem('accessToken')` | 应用启动时从持久化恢复 |
| `refreshToken` | `string \| null` | `localStorage.getItem('refreshToken')` | 同上 |
| `currentUser` | `UserProfile \| null` | `null` | 页面刷新后由 router guard 调 `fetchCurrentUser()` 恢复 |

#### Getters

| 名称 | 逻辑 |
|------|------|
| `isAuthenticated` | `!!accessToken.value` |
| `isAdmin` | `currentUser.value?.role === 'ADMIN'` |

#### Actions

**`login(userNo, password)`**
1. `authApi.login()` → 获取 `accessToken` + `refreshToken`
2. `setTokens()` → 写入 state + localStorage
3. `fetchCurrentUser()` → 填充 `currentUser`

**`logout()`**
1. `try: authApi.logout()` 通知服务端失效 refreshToken
2. `finally: clearTokens()` — 无论网络是否成功都清空本地状态

**`setTokens(access, refresh)`**
同步写入 state 和 localStorage。

**`clearTokens()`**
清空 state 三个字段及 localStorage 两项。

**`fetchCurrentUser()`**
无 token 时直接返回；否则 `GET /users/me` 并赋值 `currentUser`。

### 2.2 useNotificationStore（`src/stores/notifications.ts`）

#### State

| 字段 | 类型 | 说明 |
|------|------|------|
| `unreadCount` | `number` | 导航栏徽章数字 |
| `pollTimer` | `ReturnType<typeof setInterval> \| null` | 模块级，不暴露给外部 |

#### Actions

**`fetchUnreadCount()`**
先检查 `auth.isAuthenticated`，否则静默返回；调用 `GET /notifications/unread-count`，更新 `unreadCount`，捕获所有错误（silent fail）。

**`startPolling()`**
立即调用一次 `fetchUnreadCount()`，然后每 60,000ms 调一次。幂等：已有 timer 时不重复创建。

**`stopPolling()`**
`clearInterval` + `pollTimer = null` + `unreadCount = 0`。登出时调用。

### 3.3 全局守卫（`beforeEach`）

```
1. 若 isAuthenticated && !currentUser → fetchCurrentUser()
   失败 → clearTokens()；需权限页 → redirect /login
2. requiresAdmin → !isAuthenticated → {name:'login', query:{redirect}}
                → !isAdmin        → {name:'home'}
3. requiresAuth  → !isAuthenticated → {name:'login', query:{redirect}}
4. guest && isAuthenticated        → isAdmin ? 'admin-dashboard' : 'home'
```

表单字段：
- `<label for="userNo">` + `<input id="userNo" class="input-field">` — 学号/工号

## 8. 工具函数

### 8.1 date.ts（`src/utils/date.ts`）

| 函数 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `formatDate(date)` | ISO string | `YYYY-MM-DD` | 只取日期部分 |
| `formatDateTime(dt)` | ISO string | `YYYY-MM-DD HH:mm` | 日期+时间 |
| `fromNow(dt)` | ISO string | 相对时间（中文） | dayjs relativeTime + zh-cn |
| `today()` | — | `YYYY-MM-DD` | 今天 |
| `tomorrow()` | — | `YYYY-MM-DD` | 明天 |
| `isFuture(dateStr)` | `YYYY-MM-DD` | `boolean` | 是否为未来日期（不含今天） |

使用 dayjs 全局配置 `locale('zh-cn')` + `relativeTime` 插件，在 `date.ts` 入口处执行，全局生效。

## 9. 类型定义（`src/types/api.ts`）

### 核心接口

```typescript
// 统一响应包装
```
